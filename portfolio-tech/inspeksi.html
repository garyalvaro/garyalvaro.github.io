Vehicle Inspection System

ASP .NET MVC / C# / SQL Server / Bootstrap / JQuery

Overview

The Vehicle Inspection System (Inspeksi Armada) is a standalone application within the company's logistics ecosystem, specifically developed to manage routine vehicle inspections before and after each trip. When a vehicle is found to have issues, it is directed to the company's workshop for repairs. The inspection process is carried out by on-site inspectors using tablets, so the system is designed to be user-friendly on touchscreen devices.

Role

Fullstack Developer

Tech Stack

ASP.NET MVC (C#)
Microsoft SQL Server
Bootstrap
JQuery

Planning

The system had been in operation for quite some time, but in Q3 2023, management initiated a major system overhaul. The operational team provided a revised inspection form, which became the foundation for further development. The primary objective was to support the digital transformation of the vehicle inspection process across all company branches in Indonesia, making it more structured, user-friendly, and compliant with ISO 9001:2015 standards.

The system scope includes managing inspection criteria, recording inspection results, generating reports, and handling user account access. However, it does not yet support IoT integration, real-time notifications, or mobile app deploymentâ€”it is available solely as a web application optimized for tablet use. This project involved myself as a fullstack developer, working alongside a system analyst, one supervisor, and the management team for evaluation and approval.

Analysis

The legacy system had several limitations: a rigid input form that couldn't adapt to different types of vehicle body configurations, the absence of indicator remarks, lack of compliance with ISO 9001:2015, and an outdated interface design.

There are two primary user roles in the system:

  • Inspectors, who are responsible for conducting inspections every time a vehicle enters or leaves the depot.
  • Management, who define the inspection forms and receive summary reports from all branches.

Management's specific requests included:

  • Updating the inspection form to comply with ISO 9001:2015
  • Implementing digital signatures
  • Providing more comprehensive reporting features
  • Implementing role based access control (RBAC) and branch access

Design

I began the redesign process by rebuilding the database schema based on user needs and the new ISO-compliant inspection form. A modern admin template was selected to improve usability. Given the simplicity of the system, I skipped the mockup and flowcharting stages. However, I gave special attention to UI/UX design to ensure the interface was fully optimized for tablet displays used in the field.

Implementation

The system was built using the ASP.NET MVC framework with C#, connected to a Microsoft SQL Server database. The frontend utilized Bootstrap 4 and jQuery 3. The primary development tools were Visual Studio 2022, Visual Studio Code, and TablePlus.

As the sole developer on the project, I handled the end-to-end module development process, which included:

  • Designing and creating new database tables
  • Translating user needs into CRUD features using the MVC pattern and LINQ query builder
  • Developing responsive UI components
  • Integrating company's internal SSO login via API
  • Compressing photo attachments to reduce storage usage
  • Implementing digital signature features for inspectors and drivers
  • Adding validation rules that align with operational logistic system
  • Documenting complex code segments for better maintainability
  • Coordinating closely with the system analyst and supervisor to align outputs with user expectations

Testing

Manual testing was conducted by the system analyst after each module was developed. Any bugs found were fixed immediately. Once all features were verified, we conducted a demo session for management, incorporated their feedback, and implemented minor updates accordingly.

Deployment / Release

The system was gradually rolled out across company branches starting in early September 2023. Prior to the rollout, I handled the deployment process by publishing the project through Visual Studio 2022 and manually transferring the output files to the on-premise IIS server using Remote Desktop Connection. The domain configuration and hosting environment had already been provisioned by the company. After the system was up and running, the management team conducted training and socialization sessions with inspectors in each branch to ensure proper usage and smooth adoption.

Maintenance

I remain actively involved in system maintenance, which primarily includes fixing post-launch bugs and handling minor updates. My focus has been to ensure that vehicle inspections continue to run reliably and efficiently.
